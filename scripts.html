Scripts

Various helper scripts are included in the complete source code but not included in the python library, since they mostly deal with the dataset generation, rather than dataloading.

find_all_versions

find_all_versions.compare_replays_and_game(replays: Path, game: Path)[source]

Compare versions of replays with versions of the game verision currently present

find_all_versions.write_replay_versions(replays: Path, output: Path)[source]

Write all the game,data,build versions found in folder of replays to a csv

gen_info_header

Generates generated_info.hpp

gen_info_yaml

Generates game_info.yaml

Script to poke around and inspect the serialized replay data

inspect_replay.count(folder: Path)[source]

Count number of replays in a set of shards

make_partitions

make_partitions.main(folder: Path, output: Path, num: int)[source]

Read folder or SC2 Replay files and generate a set of files that contain an even size distribution of SC2 replay filenames

merge_info_yaml

merge_info_yaml.main(source: Path, dest: Path)[source]

Copies info from source yaml to destination yaml without duplication

replay_parallel.main(converter: Path, outfolder: Path, replays: Path, game: Path, replay_list: Path | None = None, n_parallel: int = 2)[source]

Convert replays at a target folder with n_parallel processes

replay_sql

replay_sql.create(workspace: Path = PosixPath('/home/runner/work/sc2-serializer/sc2-serializer/docs'), workers: int = 0, name: str = 'gamedata', pod_offset: int = 0)[source]

Standard creation function to create sql database from dataloader Replay database(s) are gathered from environment variable DATAPATH

replay_sql.create_individual(workspace: Path = PosixPath('.'), workers: int = 0)[source]

Create sql databases for each individual replay database. Replay database(s) are gathered from environment variable DATAPATH.

replay_sql.merge(folder: Path, target: Path)[source]

Merge all databases in directory to target
