 database.hpp
 Database framework for reading and writing serialized replay data. ReplayDatabase is templated on any type which fulfills the interface concept HasDBInterface, i.e. a specialization is defined for that type to enable reading/writing the main data and any metadata associated with the replay.
 
 instance_transform.hpp
 Instance-major transformation code for unit data which improves data compression and reduces file size from unit data by ~70%. This is done by using a lambda that sorts flattented units by their id.
 
 observer.hpp
 Contains StarCraft II game observation classes. Each variant saves observations at different rates to balance between verbosity and dataset size.
 
 replay_parsing.hpp
 Set of tools for parsing replay data into a more amenable form for machine learning. These are mainly used via their Python API.
 
 serialize.hpp
 Generic Serialisation and Deserialisation Methods. A combination of C++20 Concepts and Requires clauses are used to automatically determine how to (de)serialize data. Boost PFR is used to iterate over non-trivial structures. Underlying data is ideally trivially copyable or some range which can be copied element-by-element. This will not work properly with pointer-like objects.

 vectorize.hpp
 Automatically convert some struct S to a vector of homogeneous data type, optionally expanding enum values to one-hot encodings. Useful for writing unit data or score data to numpy arrays for ML training.
